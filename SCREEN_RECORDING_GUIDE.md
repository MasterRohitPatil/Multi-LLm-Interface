### Before You Start
- [ ] Close all unnecessary applications
- [ ] Clear browser cache and cookies
- [ ] Set browser zoom to 100%

- [ ] Turn off notifications (Windows/Mac)

- **Resolution:** 1920x1080 (Full HD)
- **Frame Rate:** 30 fps minimum (60 fps preferred)

### SCENE 1: Opening - Launch Application (5 seconds)

**Action Steps:**
1. Open browser (Chrome/Firefox)
2. Navigate to `http://localhost:5173` (or your URL)
3. Wait for application to load
4. **Pause for 2 seconds** on the workspace view

**What's on Screen:**

### SCENE 2: Setup Models (15 seconds)

**Action Steps:**

1. **Add First Pane - GPT-4**
   - Click "+" button or empty pane area
   - Click on model selector dropdown
   - Scroll to find "OpenAI"
   - Select "GPT-4" or "GPT-4 Turbo"
   - **Pause 1 second**

2. **Add Second Pane - Claude**
   - Click "+" to add another pane
   - Click model selector
   - Find "Anthropic"
   - Select "Claude 3.5 Sonnet"
   - **Pause 1 second**

3. **Add Third Pane - Gemini**
   - Click "+" for third pane
   - Click model selector
   - Find "Google"
   - Select "Gemini Pro"
   - **Pause 1 second**

**What's on Screen:**
- 3 panes side by side
- Each showing different model name
- All panes empty and ready

---

### SCENE 3: Initial Broadcast Query (45 seconds)

**Action Steps:**

1. **Click on Broadcast Bar** at bottom of screen
   - The bar should expand/highlight

2. **Select Models for Broadcast**
   - Click "Select Models" dropdown
   - Check boxes for:
     - ‚úÖ GPT-4
     - ‚úÖ Claude 3.5 Sonnet
     - ‚úÖ Gemini Pro
   - Click outside dropdown to close

3. **Type the Query** (type naturally, not too fast)
   ```
   I'm planning a destination wedding in India for 180 guests in February 2025. 
   Our budget is INR 1,00,00,000 . We want:
   
   - A palace or heritage venue with royal ambiance
   - Good weather in February
   - Easy airport access for international guests
   - Accommodation for all guests on-site or nearby
   - Traditional Indian wedding ceremonies possible
   - Beautiful photo opportunities
   
   Please suggest 3 specific venues in India with detailed pros and cons for each.
   ```

4. **Click "Broadcast to All Models"** button
   - Button should highlight/animate

5. **Watch Responses Stream In**
   - All 3 panes start showing responses simultaneously
   - Text appears in real-time
   - **Let it run for 30-40 seconds** until all responses complete
   - Scroll slowly through each pane to show content

**What's on Screen:**
- All 3 panes streaming responses
- Different text appearing in each
- Loading indicators while streaming
- Completed responses with venue suggestions

### SCENE 5: Compare Two Responses (30 seconds)

**Action Steps:**

1. **Click "Compare" button** in toolbar
   - Button should be visible at top

2. **Select First Response**
   - Click on GPT-4 pane
   - Pane should highlight/select

3. **Select Second Response**
   - Click on Claude pane
   - Both panes now selected

4. **View Comparison**
   - Diff viewer opens
   - Shows side-by-side comparison
   - Differences highlighted in colors

5. **Scroll Through Comparison**
   - Slow scroll to show differences
   - **Pause 3 seconds** on key differences
   - Highlight unique venue suggestions

6. **Close Comparison**
   - Click "X" or "Close" button
   - Return to workspace view

**What's on Screen:**
- Split-screen diff view
- Highlighted differences
- Color-coded additions/changes
- Clear comparison of venues

### SCENE 6: Deep Dive - Send Message to Another Model (45 seconds)

**Action Steps:**

1. **Select Specific Message**
   - Hover over Claude's response about "Umaid Bhawan Palace, Jodhpur"
   - Click the checkbox that appears
   - Message should highlight

2. **Click "Send To" button**
   - Button in toolbar or message menu
   - Send To menu opens

3. **Configure Send To**
   - **Target:** Select GPT-4 pane from dropdown
   - **Mode:** Select "Append" (should be default)
   - **Additional Context:** Type:
     ```
     Based on the Umaid Bhawan Palace recommendation above, please provide:
     1. Specific package details for 80 guests
     2. Month-by-month weather comparison for February
     3. Nearby accommodation options for overflow guests
     4. Transportation logistics from Jodhpur airport
     5. Recommended local wedding planners
     6. Estimated detailed cost breakdown
     ```
   - **Preserve Roles:** Keep checked ‚úÖ

4. **Click "Send Messages"**
   - Menu closes
   - Message transfers to GPT-4 pane

5. **Watch Response**
   - GPT-4 pane shows transferred message
   - New response starts streaming
   - **Wait 20-30 seconds** for complete response
   - Scroll through the detailed response

**What's on Screen:**
- Send To menu with options
- Message being transferred
- New detailed response in GPT-4
- Specific venue information

### SCENE 7: Ask Follow-up in Original Pane (20 seconds)

**What's on Screen:**

### SCENE 8: Summarize Conversation (50 seconds)

**Action Steps:**

1. **Select Multiple Messages**
   - Click checkbox on Claude's original response
   - Click checkbox on Claude's comparison response
   - Click checkbox on GPT-4's detailed breakdown
   - **3 messages selected** (counter should show)

2. **Click "Send To" button**
   - Send To menu opens

3. **Configure Summarization**
   - **Target:** Select Gemini Pro pane
   - **Mode:** Select "Summarize conversation" (radio button)
   - **Summary Instructions:** Type:
     ```
     Create an executive summary for decision-making that includes:
     - Top 2 recommended venues with key differentiators
     - Critical pros and cons for each
     - Budget breakdown and value assessment
     - Weather and logistics summary
     - Final recommendation with reasoning
     ```
   - **Additional Context:** Type:
     ```
     This summary will be shared with family members to help make 
     the final venue decision.
     ```

4. **Click "Generate Summary"**
   - Button shows "Generating Summary..." with loading spinner
   - **Wait 5-10 seconds**
   - Summary appears in Gemini pane as user message
   - Gemini starts responding to the summary
   - **Wait 20-25 seconds** for complete response

5. **Scroll Through Summary**
   - Show clean, concise summary
   - Highlight key decision points

**What's on Screen:**
- Send To menu with summarize mode
- Loading state during generation
- Clean summary in Gemini pane
- Gemini's response to summary

## üé¨ Pro Tips for Great Recording

### Mouse Movement
- Move cursor smoothly and deliberately
- Don't move too fast
- Pause cursor on important elements
- Use cursor to guide viewer's eye

### Clicking
- Click clearly and deliberately
- Pause briefly after clicks
- Let animations complete
- Don't double-click unless necessary

### Scrolling
- Scroll slowly and smoothly
- Pause at important content
- Don't scroll too far too fast
- Use smooth scrolling if available

### Timing
- Let responses complete fully
- Don't rush between actions
- Pause for 1-2 seconds at key moments
- Give viewers time to read

### Common Mistakes to Avoid
- ‚ùå Moving mouse too fast
- ‚ùå Clicking before animations complete
- ‚ùå Scrolling too quickly
- ‚ùå Not showing complete responses
- ‚ùå Rushing through features
- ‚ùå Forgetting to show important UI elements

## üîß Troubleshooting During Recording

### If API Fails
- **Don't panic** - pause recording
- Check API keys
- Restart backend if needed
- Resume recording from last good point

### If Response is Slow
- **Keep recording** - shows real-world usage
- Can speed up in editing
- Shows patience required

### If You Make a Mistake
- **Option 1:** Pause, fix, resume from that point
- **Option 2:** Continue and edit out later
- **Option 3:** Restart that scene

### If UI Glitches
- Refresh page (if session persists)
- Restart recording from last scene
- Have backup recording ready
